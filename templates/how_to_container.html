Singularity

Singularity is a tool for building containers for use on multi-user computing clusters. Typically, people convert Docker containers to Singularity, as Docker is far more common.

  • Code at github
  • Singularity docs
  • Docker docs

Singularity 3.4.2 is installed on each node, as is docker 19.03.5. You must have special permissions to run docker, but anyone can use singularity. Read the singularity docs linked above for more, and the docker docs for a stronger understanding of what containers are doing. For simple examples, see the Singularity recipes on jjm15, at /var/lib/singularity/. These recipes either build containers directly in Singularity or download a docker container and convert it to Singularity; the FreeSurfer recipe copies a license file into the container, too.

If you'd like a quick demonstration, download a demo container like so:

singularity pull docker://godlovedc/lolcow

You can work inside of the container like so (type exit<Enter> to leave):

singularity shell lolcow_latest.sif

Or just run the default, or code from it like so:

./lolcow_latest.sif
singularity exec lolcow_latest.sif cowsay "Hi, there!"
